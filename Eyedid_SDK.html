EyeDID SDK 웹 통합 가이드

MediaPipe FaceMesh 기반 프로토타입에서 EyeDID(구 SeeSo) SDK로 마이그레이션하기 위한 단계별 가이드입니다. API 사용법, 캘리브레이션 흐름, 그리고 눈길 프로젝트에 맞는 구현 전략을 다룹니다.

STEP 0 MediaPipe vs EyeDID 비교

현재 프로토타입은 MediaPipe FaceMesh 468 랜드마크에서 iris 중심을 추출하고, 직접 구현한 2차 다항식 회귀로 화면 좌표를 매핑하고 있습니다. EyeDID SDK로 전환하면 다음과 같은 이점이 있습니다.

항목 MediaPipe (현재) EyeDID SDK
시선 추적 방식 iris 좌표 → 직접 회귀 매핑 딥러닝 기반 Gaze Estimation (화면 좌표 직접 출력)
캘리브레이션 직접 구현 (16점 + 검증 4점) SDK 내장 (1/5/6점 모드 선택 가능)
정확도 환경에 따라 편차 큼 (60~150px) 약 2~3cm 오차 (일반 웹캠 기준)
눈 깜빡임 감지 EAR 비율 직접 계산 BlinkInfo로 자동 제공
사용자 상태 없음 Attention Score, Drowsiness 등 내장
필터링 직접 구현 (Median+EMA+Velocity) OneEuroFilter 내장 제공
비용 무료 (오픈소스) 개발용 무료 / 프로덕션 유료
핵심: 현재 프로토타입에서 직접 구현한 iris 추출, 다항식 회귀, 캘리브레이션 그리드, 검증 로직이 모두 SDK 한 줄로 대체됩니다. 여러분의 앱 로직(읽기 추적, 집중도 분석, UI)에만 집중할 수 있게 됩니다.

STEP 1 라이선스 키 발급

EyeDID SDK는 인증 기반으로 동작합니다. 시작 전 반드시 라이선스 키를 발급받아야 합니다.

1
manage.seeso.io 접속 → 회원가입

VisualCamp 콘솔에 가입합니다. 구글 계정으로 빠르게 가입 가능합니다.

2
콘솔 → License Keys → Development Key 생성

개발(Development) 키는 무료이며, 도메인 제한 없이 테스트할 수 있습니다. 프로덕션 키는 별도 요청이 필요합니다.

3
키를 안전하게 보관

발급받은 키를 환경 변수나 설정 파일에 저장합니다. 클라이언트 사이드 코드에 직접 노출되지만, Development 키는 이 방식이 일반적입니다.

주의: SDK 인증에는 인터넷 연결이 필수입니다. 초기화 시 EyeDID 서버에서 세션을 발급받으며, 오프라인 환경에서는 작동하지 않습니다.

STEP 3 GazeTracker 초기화

EasySeeSo는 SeeSo SDK를 간편하게 사용할 수 있는 래퍼 클래스입니다. init() 호출 시 라이선스 키 인증 → 카메라 접근 → GazeTracker 생성이 순차적으로 이루어집니다.

javascript — 초기화 
const licenseKey = 'YOUR_LICENSE_KEY';

// EasySeeSo 인스턴스 생성
const seeso = new EasySeeSo();

// 초기화 (비동기)
async function initEyeTracking() {
  try {
    await seeso.init(
      licenseKey,
      // 시선 콜백 — 핵심!
      onGaze,
      // 디버그 콜백 (선택)
      onDebug
    );
    console.log('✅ EyeDID 초기화 성공');
  } catch(e) {
    console.error('❌ 초기화 실패:', e);
  }
}

// 시선 데이터 콜백
function onGaze(gazeInfo) {
  if (gazeInfo.trackingState === TrackingState.SUCCESS) {
    const x = gazeInfo.x;  // 화면 X 좌표 (px)
    const y = gazeInfo.y;  // 화면 Y 좌표 (px)
    // → 여기서 시선 처리 로직 실행
  }
}

function onDebug(FPS, accuracy) {
  console.log(`FPS: ${FPS}, Accuracy: ${accuracy}`);
}
MediaPipe 대비 변화: 기존에 FaceMesh 초기화 → Camera 시작 → onResults 콜백에서 iris 추출 → 직접 predict 하던 과정이 seeso.init(key, onGaze) 한 줄로 대체됩니다.

STEP 4 시선 추적 시작

초기화 성공 후 startTracking()을 호출하면 웹캠 기반 시선 추적이 시작됩니다. 매 프레임마다 onGaze 콜백이 호출됩니다.

javascript — 추적 시작/종료 
// 추적 시작
seeso.startTracking();

// 추적 종료
seeso.stopTracking();

// ─────────────────────────────────────────────
// onGaze 콜백에서 받는 gazeInfo 객체 구조:
// ─────────────────────────────────────────────
function onGaze(gazeInfo) {
  // gazeInfo.x              → 화면 X 좌표 (px)
  // gazeInfo.y              → 화면 Y 좌표 (px)
  // gazeInfo.trackingState  → SUCCESS | LOW_CONFIDENCE | UNSUPPORTED
  // gazeInfo.eyeMovementState → FIXATION | SACCADE | UNKNOWN
  // gazeInfo.timestamp      → 밀리초 타임스탬프

  if (gazeInfo.trackingState === TrackingState.SUCCESS) {
    // 높은 신뢰도 — 정밀 추적에 사용
    moveGazeDot(gazeInfo.x, gazeInfo.y);
    detectGazedWord(gazeInfo.x, gazeInfo.y);
  }
  else if (gazeInfo.trackingState === TrackingState.LOW_CONFIDENCE) {
    // 낮은 신뢰도 — 대략적 위치만 참고
    moveGazeDot(gazeInfo.x, gazeInfo.y);
  }
  else {
    // UNSUPPORTED — 얼굴 미감지
    showFaceMissing();
  }
}
TrackingState 주의: SUCCESS와 LOW_CONFIDENCE 모두 유효한 좌표를 반환합니다. 읽기 추적처럼 정밀도가 필요한 경우 SUCCESS만 사용하고, 단순 시선 표시는 둘 다 활용하세요.

STEP 5 캘리브레이션

EyeDID는 두 가지 캘리브레이션 방식을 제공합니다. 눈길 프로젝트에는 커스텀 캘리브레이션이 적합합니다.

방법 A — Quick 캘리브레이션 (외부 페이지)

SDK가 제공하는 별도 캘리브레이션 페이지로 리다이렉트합니다. 간단하지만 UX 커스터마이즈가 제한적입니다.

javascript — Quick 방식 
// 캘리브레이션 페이지로 리다이렉트
EasySeeSo.openCalibrationPage(
  licenseKey,
  'user-id-123',         // 사용자 식별자
  'https://myapp.com/read', // 완료 후 리다이렉트 URL
  5                         // 캘리브레이션 포인트 수 (1 or 5)
);

// ─ 리다이렉트된 페이지에서 캘리브레이션 데이터 파싱 ─
function loadCalibrationData() {
  const href = window.location.href;
  const decoded = decodeURI(href);
  const qs = decoded.split('?')[1];
  if (!qs) return;
  const json = qs.slice('calibrationData='.length);
  seeso.setCalibrationData(json);
}

방법 B — 커스텀 캘리브레이션 (권장)

앱 내에서 직접 캘리브레이션 UI를 구현합니다. 눈길의 기존 디자인 시스템을 유지할 수 있습니다.

javascript — 커스텀 캘리브레이션 (핵심 코드) 
// 캘리브레이션 시작 (5점 모드)
const calibrationPoints = 5; // 1, 5, 6 중 선택

seeso.startCalibration(calibrationPoints);

// ──────────────────────────────────────────────
// 콜백 1: 다음 캘리브레이션 포인트 좌표 전달
// SDK가 "이 좌표를 사용자에게 보여주세요"라고 알려줌
// ──────────────────────────────────────────────
seeso.setOnCalibrationNextPoint(function(x, y) {
  // 1) 해당 좌표에 캘리브레이션 점(dot) UI를 표시
  showCalibDot(x, y);

  // 2) 사용자가 점을 바라볼 시간을 준 뒤 샘플 수집 시작
  setTimeout(() => {
    seeso.startCollectSamples();
  }, 1000); // 1초 대기
});

// ──────────────────────────────────────────────
// 콜백 2: 진행률 (0.0 ~ 1.0)
// ──────────────────────────────────────────────
seeso.setOnCalibrationProgress(function(progress) {
  updateCalibRing(progress); // 기존 링 애니메이션 재활용
});

// ──────────────────────────────────────────────
// 콜백 3: 캘리브레이션 완료 → 데이터 저장
// ──────────────────────────────────────────────
seeso.setOnCalibrationFinished(function(calibrationData) {
  console.log('✅ 캘리브레이션 완료');

  // calibrationData를 저장해두면 재시작 시 재캘리브 불필요
  localStorage.setItem('eyedid_calib', calibrationData);

  // → 읽기 화면으로 전환
  proceedToRead();
});
큰 이점: 캘리브레이션 데이터를 localStorage에 저장하고 seeso.setCalibrationData(data)로 복원할 수 있습니다. 사용자가 다시 접속해도 재캘리브레이션 없이 즉시 사용 가능합니다!

기존 눈길 캘리브레이션과의 매핑

눈길 기존 (MediaPipe) EyeDID SDK 대응
16점 그리드 + 클릭 수집 startCalibration(5) — 5점이면 충분
SETTLE_MS + SAMPLE_MS 타이밍 onCalibrationNextPoint → setTimeout → startCollectSamples
링 애니메이션 onCalibrationProgress로 동일하게 구현
fitModel() 다항식 회귀 삭제 — SDK 내부에서 자동 처리
검증(Validation) 4점 선택 — 자체 검증 로직은 유지 가능 (onGaze 좌표 vs 표시 점 비교)

STEP 6 필터링 & 후처리

EyeDID SDK는 OneEuroFilter를 내장 제공합니다. 기존 눈길의 Median+EMA+Velocity damping 파이프라인을 대체할 수 있습니다.

javascript — OneEuroFilter 사용 
// seeso 패키지에서 OneEuroFilter import
import { OneEuroFilter } from 'seeso';

// 필터 생성 (2차원: x, y)
const gazeFilter = new OneEuroFilter(2);

function onGaze(gazeInfo) {
  if (gazeInfo.trackingState === TrackingState.SUCCESS) {
    // 필터 적용
    if (gazeFilter.filterValues(gazeInfo.timestamp, [gazeInfo.x, gazeInfo.y])) {
      const [fx, fy] = gazeFilter.getFilteredValues();
      moveGazeDot(fx, fy);
      detectGazedWord(fx, fy);
    }
  }
}
기존 코드 제거 대상: medianOf(), filterCoords(), EMA 로직, Velocity damping, bufX/bufY 버퍼 — 모두 OneEuroFilter가 대체합니다.

STEP 8 전체 코드 구조

마이그레이션 후 핵심 코드의 전체 구조입니다. 기존 1,175줄에서 약 300~400줄로 대폭 줄어듭니다.

javascript — 마이그레이션 후 핵심 구조 
'use strict';
// ════════════════════════════════════════════
//  눈길 — EyeDID SDK 기반 시선 추적
// ════════════════════════════════════════════

const LICENSE_KEY = 'YOUR_LICENSE_KEY';
let seeso;
let gazeFilter;

// ── State (기존 S 객체에서 SDK 관련 항목 제거) ──
const S = {
  calibDone: false,
  readStart: null,
  onFrames: 0, totFrames: 0,
  gazedWords: new Set(),
  totalWords: 0,
  passageIdx: 0,
  stabBuf: [],
};

// ════════════════════════════════════════════
//  1. 초기화
// ════════════════════════════════════════════
async function initEyeTracking() {
  seeso = new EasySeeSo();
  gazeFilter = new OneEuroFilter(2);

  try {
    await seeso.init(LICENSE_KEY, onGaze, onDebug);
    setStatus('SDK 초기화 완료', 'on');
    setStep(1);  // 캘리브레이션 화면으로
    startCalibration();
  } catch(e) {
    setStatus('초기화 실패: ' + e, 'err');
  }
}

// ════════════════════════════════════════════
//  2. 캘리브레이션 (커스텀 UI)
// ════════════════════════════════════════════
function startCalibration() {
  // 저장된 캘리브레이션 데이터가 있으면 재사용
  const saved = localStorage.getItem('eyedid_calib');
  if (saved) {
    seeso.setCalibrationData(saved);
    proceedToRead();
    return;
  }

  seeso.startTracking();
  seeso.startCalibration(5);

  seeso.setOnCalibrationNextPoint((x, y) => {
    showCalibDot(x, y); // 기존 UI 재활용
    setTimeout(() => seeso.startCollectSamples(), 1000);
  });

  seeso.setOnCalibrationProgress((progress) => {
    updateCalibRing(progress);
  });

  seeso.setOnCalibrationFinished((data) => {
    localStorage.setItem('eyedid_calib', data);
    proceedToRead();
  });
}

// ════════════════════════════════════════════
//  3. 시선 처리 (핵심 콜백)
// ════════════════════════════════════════════
function onGaze(gazeInfo) {
  if (gazeInfo.trackingState !== TrackingState.SUCCESS) return;

  // OneEuroFilter 적용
  if (!gazeFilter.filterValues(gazeInfo.timestamp, [gazeInfo.x, gazeInfo.y])) return;
  const [gx, gy] = gazeFilter.getFilteredValues();

  // Gaze dot 이동
  gazeDot.style.left = gx + 'px';
  gazeDot.style.top  = gy + 'px';

  // Trail
  trail.push({ x: gx, y: gy, t: Date.now() });
  if (trail.length > 50) trail.shift();
  drawTrail();

  // 읽기 추적 (기존 로직 그대로)
  if (S.calibDone) {
    S.totFrames++;
    detectGazedWord(gx, gy);
    updateProgBar(gy);
    updateStabUI();
  }
}

다음 단계: 프로젝트 확장

SDK 마이그레이션이 완료되면, 기능구조도에 명시된 핵심 기능들을 추가해 나갈 수 있습니다.

A
집중도 분석 엔진 구현

Fixation Duration, Regression Count, Gaze Dispersion을 onGaze 콜백 데이터로 계산. eyeMovementState가 FIXATION/SACCADE를 자동 구분해줍니다.

B
집중 Mode (블러 효과)

시선이 위치한 문단만 선명하게, 나머지는 블러/채도 저하. onGaze에서 현재 시선 영역을 판별하여 CSS 필터를 동적 적용.

C
Attention Score 활용

EyeDID의 UserStatus 기능으로 집중도를 자동 측정. 임계값 이하 시 알림을 트리거하는 로직 추가.
