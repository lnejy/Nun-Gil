03/16 캡스톤 디자인 회의록

일시: 2025년 3월 16일 (일) 오후 2시 ~ 5시 30분
장소: 한서대학교 공학관 404호 회의실
참석자: 김민준(팀장), 이수아, 박지훈, 최유진, 정다은
서기: 이수아
프로젝트명: 눈길 — 시선 추적 기반 지능형 문서 학습 보조 시스템

1. 회의 개요 및 목적

본 회의는 캡스톤 디자인 프로젝트 '눈길'의 2차 정기 회의로, 지난 1차 회의에서 결정된 기획안을 토대로 각 파트별 진행 상황을 점검하고, 향후 3주간의 개발 로드맵을 구체화하기 위해 소집되었다. 특히 아이트래킹 모듈의 기술 선정 문제와 프론트엔드 뷰어 UI 설계 방향에 대해 집중적으로 논의하였다.

팀장 김민준은 회의 시작에 앞서 현재 프로젝트 일정이 당초 계획 대비 약 5일 지연된 상태임을 공유하며, 이번 회의를 통해 핵심 기술 스택을 확정하고 담당자별 마감일을 재설정하는 것을 목표로 삼겠다고 밝혔다. 또한 지도 교수님의 중간 발표 피드백을 바탕으로, 사용자 경험(UX) 측면에서 보다 직관적인 인터페이스 설계가 필요하다는 점을 강조하였다.

2. 파트별 진행 현황 보고

2-1. 아이트래킹 모듈 (담당: 박지훈)

박지훈은 지난 2주간 WebGazer.js와 GazeRecorder API를 비교 분석한 결과를 공유하였다. WebGazer.js의 경우 오픈소스 기반으로 커스터마이징이 자유롭다는 장점이 있으나, calibration 과정이 복잡하고 CPU 사용률이 높아 저사양 기기에서 불안정한 문제가 있었다.

테스트 결과, 일반 웹캠 환경에서 WebGazer.js의 평균 시선 오차는 약 98픽셀, GazeRecorder는 약 62픽셀로 측정되었으며, 이는 A4 용지 기준 약 2~3줄 범위의 정확도에 해당한다.

2-2. 문서 뷰어 UI (담당: 이수아, 최유진)

이수아는 현재까지 구현된 PDF 뷰어 인터페이스의 데모를 시연하였다. 기본적인 문서 표시 기능, 단어 클릭 시 정의 팝업, 폰트 크기 및 종류 변경 기능, 메모장 사이드바가 완성된 상태이며, 아이트래킹 패널의 활성화/비활성화 UI도 구현되었다고 보고하였다.

최유진은 5명의 베타 테스터를 대상으로 진행한 프로토타입 테스트에서, 참여자의 80%가 현재 인터페이스의 직관성에 만족감을 표했으나, 60%가 "읽다가 어디까지 읽었는지 표시가 되면 좋겠다"는 의견을 제시하였다.

2-3. 백엔드 및 데이터 처리 (담당: 정다은)

정다은은 Flask 기반의 백엔드 API 서버 구축 현황을 공유하였다. 현재 PDF 파일 업로드 및 텍스트 추출 기능(PyMuPDF 활용)이 완성되었으며, 단어별 빈도 분석 및 형태소 분석(KoNLPy Okt 모듈 사용)도 기본 동작이 확인되었다.

또한 GPT-4o API를 활용한 문서 요약 기능의 초기 프로토타입이 완성되었다. 현재 평균 응답 시간이 약 4~7초로, 스트리밍 응답 방식으로의 전환을 검토 중이라고 밝혔다.

3. 주요 의사결정 사항

3-1. 기술 스택 최종 확정

프론트엔드는 바닐라 JavaScript와 HTML/CSS를 기반으로 하되, 복잡한 상태 관리가 필요한 경우 Alpine.js 도입을 검토하기로 하였다. 백엔드는 Python 3.11 기반의 Flask 프레임워크를 유지하며, 데이터베이스는 SQLite를 1차 적용 후 필요 시 PostgreSQL로 마이그레이션하는 방향으로 결정되었다.

3-2. 북마크 기능 설계 방향 결정

팀원들은 기존 PDF 뷰어들과의 차별화를 위해, 단순한 페이지 북마크 방식이 아닌 문장 단위 인덱스 탭 방식을 채택하기로 하였다. 뷰어 왼쪽에 색상이 구분된 탭이 생성되어 사용자가 시각적으로 북마크 위치를 파악할 수 있도록 설계된다.

3-3. 중간 발표 준비 방향

4월 셋째 주로 예정된 중간 발표에 대한 준비 방향이 논의되었다. 팀장 김민준은 "왜 이 기능이 학습자에게 필요한가"를 중심으로 발표 스크립트를 구성하겠다고 밝혔다.

4. 개발 스프린트 계획 (3/17 ~ 4/6)

Week 1 (3/17 ~ 3/23): 핵심 기능 완성

  • 북마크 기능 구현 및 테스트 — 이수아, 최유진
  • 아이트래킹 프로토타입 정확도 개선 — 박지훈
  • 요약 API 스트리밍 방식 전환 벤치마크 — 정다은
  • 마인드맵 뷰 기본 레이아웃 구현 — 이수아

Week 2 (3/24 ~ 3/30): 통합 및 테스트

  • 프론트엔드-백엔드 API 연동 1차 통합 — 전원
  • 아이트래킹 모듈 뷰어 연동 — 박지훈, 이수아
  • 한자 혼용 문서 OCR 전처리 개선 — 정다은
  • 내부 1차 QA 진행 — 최유진 주도

Week 3 (3/31 ~ 4/6): 데모 준비 및 발표 자료 제작

  • 데모 영상 시나리오 작성 및 촬영 — 최유진, 정다은
  • 발표 PPT 초안 작성 — 김민준
  • 성능 지표 정리(시선 추적 정확도, 응답 시간 등) — 박지훈
  • 사용자 테스트 2차 진행 (외부 5명) — 최유진

6. 기타 논의 사항

6-1. 협업 도구 정비

현재 팀 협업 도구로 Notion(문서), GitHub(코드), KakaoTalk(실시간 소통)을 병행 사용 중이다. 최유진은 Notion의 데이터베이스 기능을 활용하여 스프린트 진행 현황을 시각화하는 대시보드를 만들겠다고 제안하였으며, 팀원 모두가 동의하였다.

6-2. 저작권 및 라이선스 검토

정다은은 프로젝트에서 사용 중인 오픈소스 라이브러리들의 라이선스를 검토한 결과를 공유하였다. PyMuPDF는 AGPL-3.0 라이선스를 따르므로, 상업적 배포 시 소스코드 공개 의무가 발생할 수 있음을 지적하였다.

6-3. 멘토링 일정 조율

지도 교수님과의 월간 멘토링이 3월 28일 오후 4시로 예정되어 있음을 팀장 김민준이 재공지하였다. 회의 전까지 각 파트의 진행 현황을 1페이지 분량의 요약 슬라이드로 정리하여 팀 공유 폴더에 업로드해 줄 것을 요청하였다.

7. 회의 결론 및 다음 회의 예고

이번 회의를 통해 프로젝트의 핵심 기술 스택이 확정되었으며, 북마크 기능의 Phase 1 포함이 결정되었다. 또한 향후 3주간의 스프린트 계획과 담당자별 실행 항목이 명확하게 정리되었다.

다음 정기 회의는 2025년 3월 30일(일) 오후 3시, 같은 장소에서 개최될 예정이다.
